## Task 4 - Path Parameter vs Query Parameter

### Path Parameter

A path parameter identifies a specific resource and is part of the URL path.

Example:

GET /projects/5

Here, 5 is the path parameter representing the project ID.

### Query Parameter

A query parameter is used to filter, sort, or search data. It appears after the `?` in the URL.

Example:

GET /projects?status=completed

Here, status=completed filters the projects to show only completed ones.
